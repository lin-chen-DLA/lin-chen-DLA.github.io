Introduction to Artifical Neural Networks with Kera
¶

Perceptron¶

  • Based on artifical neurons, the inputs and output are numbers, and each input connection is associated with a weight
  • Perceptron is composed of a single layer of Threshold Logic Unit (TLUs), with each TLU connected to all the inputs
  • Fully Connected Layer, or Dense Layer
    • All the neurons in a layer are connected to every neuron in the previous layer
  • The decision boundary is linear
  • Hebb's rule
    • fed one training instance at a time
    • for every output neuron that produced a wrong prediction, reinforces the connection weights from the inputs that would have contriubted to the correct prediction
  • Do not output a class probability; rather, Perceptron classifier makes predictions based on a hard threshold
  • Cannot solve Exclusive OR as any other linear classification models

Multilayer Perceptron (MLP)¶

  • Stack multiple Perceptrons
  • One input layer, one or more hidden layers of TLUs, one output layer
  • Gradient Descent steps
    • for each training instance, make a prediction and measures the error
    • goes through each layer in reverse to measure the error contribution from each connection
    • tweaks the connection weights to reduce the error
  • Keras model can be used just like regular layers
  • Calling the fit() method again, Keras will continue traiing where it left off
  • Transfer Learning
    • resue the lower layers of the previous network

Regression MLPs¶

  • For output neurons
    • No activation function, free to output any range of values
    • ReLU, the output is always positive
    • Softplus, $softplus(z) = log(1+exp(z))$, a smooth variant of ReLU, close to 0 when z is negative, close to z when z is positive
    • Logistic function or hyperbolic tangent, then scale to the appropriate range, within a given range of values
  • Loss function
    • Mean squared error (MSE)
    • Mean absolute error (MAE), if have a lot of outliers
    • Huber loss, a combination of both
  • Architecture
    • input neurons, one per input feature
    • hidden layers, depends on the problems, typically 1 to 5
    • neurons per hidden layer, depend on the problem, typically 10 to 100
    • output neurons, 1 per prediction dimension
    • Hidden activation, ReLU or SELU
    • Output activation, none, or ReLU/softplus
    • Loss function, MSE, MAE, or Huber

Classification MLPs¶

  • Output is between 0 and 1, which you can interpret as the estimated probability of the positive class
  • Dedicate one output neuron for each positive class, the probabilities do not necessarily add up to 1
  • Softmax activaton is used as the activation function for the output layer to ensure estimated probabilities are betweenn 0 and 1, and they add up to 1
$$ \sigma(z)_{i} = \frac{e^{z_{i}}}{\Sigma^{k}_{j=1} e^{z_{j}}} $$
  • Architecture
    • input and hidden layers, same as regression
    • # output neurons 1 per class
    • output layer activation
      • binary, logistic
      • multilabel binary, logistic
      • multiclass, softmax, which makes all estimated probabilites are between 0 and 1 and that they add up to 1
    • Loss function, cross-entropy

Subclassing API¶

  • Model's architecture is hidden within the call() method
  • Cannot easily inspect it with Keras
  • Lose the information about how layers are connected to each other
In [141]:
class WideAndDeepModel(keras.models.Model):
    def __init__(self, units=30, activation="relu", **kwargs):
        """Create layters"""
        super().__init__(**kwargs)
        self.hidden1 = keras.layers.Dense(units, activation=activation)
        self.hidden2 = keras.layers.Dense(units, activation=activation)
        self.main_output = keras.layers.Dense(1)
        self.aux_output = keras.layers.Dense(1)
        
    def call(self, inputs):
        """Create architecture and conduct operations"""
        input_A, input_B = inputs
        #print(len(inputs))
        hidden1 = self.hidden1(input_B)
        hidden2 = self.hidden2(hidden1)
        concat = keras.layers.concatenate([input_A, hidden2])
        main_output = self.main_output(concat)
        aux_output = self.aux_output(hidden2)
        return main_output, aux_output

model = WideAndDeepModel(30, activation="relu")

def build_model(n_hidden=1, n_neurons=30, learning_rate=3e-3, input_shape=[8]):
    model = keras.models.Sequential()
    model.add(keras.layers.InputLayer(input_shape=input_shape))
    for layer in range(n_hidden):
        model.add(keras.layers.Dense(n_neurons, activation="relu"))
    model.add(keras.layers.Dense(1))
    optimizer = keras.optimizers.SGD(lr=learning_rate)
    model.compile(loss="mse", optimizer=optimizer)
    return model

Hyperparameter Selection¶

    1. Learning rate
      • the optimal learning rate is about half of the maximum learning rate which makes the training algorithm diverges
      • optimal learning rate depends on the other hyperparameters, especially the batch size
    1. Optimizer
    1. Number of layers
      • one hidden layer having enough neurons can theoretically model even the most complex functions
      • lower hidden layers model low-level structures, e.g. line segments of various shapes and orientations
      • intermediate hidden layers combine low-level structures to model intermediate-level structures, e.g. squares, circles
      • highest hidden layers and the output layer conbine intermediate structures to model high-level structures, e.g. faces
      • ramp up the number of hidden layers until overfitting the training set
      • Reusing parts of apretrained state-of-the-art network will be faster and reuqures much less data
    1. Number of neurons per layer, activation function
      • form a pyramid, with fewer and fewer neurons at each layer
      • using the same number of neurons in all hidden layers performs well in most cases, or even better
      • sometimes help to make the first hidden layer bigger than the others
      • pick more layers and neurons than you need, then use ealy stopping and other regularization techniques to prevent it from overfitting
      • in general, increase the number of layers instead of the number of neurons per layer
    1. Activation functions
      • ReLU is a good default for all hidden layers
      • depends on tasks for the output layer
    1. Batch size
      • use a large batch size, use learning rate warmup, if unstable or the final performance is disppointing, use small batch size, such as 32 tweeted by Yann LeCun
    1. Number of iterations
      • Does not actually need to be tweaked, just use early stopping instead
