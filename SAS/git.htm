Github
Create Profile
  • Allows the access to Github from SAS Viya

  • Github Side
    • Username
      • Click the Avatar on the top-right, click Your profile
      • On the profile page, your email and username is under the Avatar
    • Token
      • Click the Avatar on the top-right, click Settings
      • Click Developer settings on the bottom-left
      • Click Personal access tokens, then choose Tokens (classic)
      • Generate a token and save it
  • SAS Viya Side
    • Click Options, then choose Manage Git Connection, select Profiles
    • Click ✚ to add a git profile
      • Choose HTTPS on the top-right of the popup window
      • Profile name, a user-defined functional name
      • User name, the username on Github
      • Email, the email on Github
      • Password, the token created on Github
    • Start Project with Git Enabled

  • Option 1
    • Github Side
      • Create an empty repo or locate an existing repo
      • On the Code page, click the green Code button, copy the HTTPS URL of the repo
    • SAS Viya
      • Create a folder on SAS to store the project code
      • Open Clone a Repository window
        • Click Git Repositories, then click Clone
        • Or click Options, choose Manage Git Connections, select Repositories, click ✚
      • Use the HTTPS URL of the repo, specify the created project folder on SAS, choose the created profile, click Clone

  • Option 2
    • Github Side
      • Create an empty repo and copy the HTTPS URL of the repo
    • SAS Viya
      • Create a folder on SAS and create project code
      • Right-click the project folder, then choose Git initialize
      • On the Git Repositories page, choose the project, click Push on the right window to add the Github repo url
    • Commit
  • Edit and save code files to the project folder
  • In the Unstaged Changes window of the Git Repositories page, use ✚ to move files which need to commit to staged status
  • In the Staged Changes window of the Git Repositories page, type in commit comment and click Commit
  • Click Push to push project to the repo on Github
    • Branch
  • Create a branch
    • On the Git Repositories page, open commit history
    • Navigate to a commit you would create a branch, right-click and select Create new branch
  • Checkout a branch
    • Select the target branch on the right window of the Git Repositories page
    • Or on the left window of the Git Repositories page, right-click a branch of a project, then choose Checkout branch_name
  • Merge branches
    • Checkout to the target branch you want to merge into
    • On the Git Repositories page, open commit history, right-click the branch you want to merge to the target, choose Merge into target_branch_name
    • Push to Github
  • On the Git Repositories page, select the branch you would push to the remote repo
  • Click Push to push project to the repo on Github
    • First time push will ask adding the HTTPS url of the remote repo
