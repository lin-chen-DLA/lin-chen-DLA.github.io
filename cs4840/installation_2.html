1. Install Node.js

The entire MEAN development environment relies on a JavaScript package manager and there are two choices -- npm and yarn. In this class, as a matter of fact, it does not matter which one to use because both work just fine. Most of the times, I myself will use npm.

FYI: yarn is not an attempt to replace npm completely. Yarn is only a new CLI client that fetches modules from the same npm registry. Nothing about the registry itself will change — you’ll still be able to fetch and publish packages as normal.

So, to get yarn (optional), we need npm , which is part of node (Node.js). To get node, we need nvm (Node Version Manager).

  1. To install Node.js, begin by running this command to download Node Version Manager (nvm). (nvm is a simple Bash shell script that is useful for installing and managing Node.js versions. For the latest information, see Node Version Manager on the GitHub website.)

    curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.34.0/install.sh | bash

    To start using nvm, either close the terminal session and start it again, or source the ~/.bashrc file that contains the commands to load nvm.

    . ~/.bashrc

    In order to be certain that the latest Node.js is used when you login to your computer, open the file ~/.bash_profile (using vim or any editor) and add the command . ~/.bashrc on a separate line at the bottom of the file.

  2. Run this command to install the latest version of Node.js.

    nvm install node

    Confirm Node.js is already installed by running 

    node --version

    The latest version of Node.js is v12.7.0 as of August 2019.

  3. Node.js comes with npm - a package manager primarily for JavaScript and the world's largest software registry.

    Make sure you run the following to use the latest version of npm (at least 6.10):

    npm install -g npm@latest

  4. (Optional) However, as I said earlier, there is an arguably more reliable, more secure, and faster alternative: yarn. To install yarn, run the following command.

    npm install -g yarn

And then, in general, you need to ensure you have the latest security updates and bug fixes for the software installer on your OS, which could be one the following

  • brew (or homebrew) for Mac
  • apt-get for Debian or Ubuntu Linux
  • yum for for Fedora or Red Hat Linux

2. Frontend Installation

A modern build pipeline for React + Redux web application development typically consists of:

  • A package manager, such as yarn or npm. It lets you take advantage of a vast ecosystem of third-party packages, and easily install or update them.

    This is already taken care of in step 1 above.

  • A bundler, such as webpack or Browserify. It lets you write modular code and bundle it together into small packages to optimize load time.

  • A transpiler such as Babel. It lets you write modern JavaScript code that still works in older browsers that are only capable of handling vanilla JavaScript.

3. Backend Installation

A modern build pipeline for Node + Express + MongoDB web API server development typically consists of:

  • A package manager, such as yarn or npm. It lets you take advantage of a vast ecosystem of third-party packages, and easily install or update them.

    This is already taken care of in step 1 above.

  • NodeJS server, a JavaScript runtime.

  • MongoDB, a Database that persists users' data.

  • Optionally, Express, a library that runs in the Node runtime to make dealing with HTTP traffic easier.

  • Optionally, MongooseJS, a library that makes dealing with MongoDB transactions easier.
