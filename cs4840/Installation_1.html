2. Systems

In the order of preference, as far as the systems are concerned, you have the following two choices:

2.1. Your own computer (Preferred)

There is no doubt that a Linux or Mac computer is much more preferred than a Windows one. Among many other reasons, Linux/Mac has a far better command-line terminal – an indispensable tool when building Full-Stack Web projects.

So, if you are a Linux or Mac user, move on to Part 3.

For the Windows users, here is the deal – you either install a Linux OS (which is super easy nowadays) or opt to use Cloud9 (not recommended though).

2.2. AWS Cloud9 -- a cloud-based IDE

Warning: To use AWS C9, all you need is a browser. However, I am not a big fan of it because my experience tells me that its performance highly relies on a number of factors that are beyond our control such as network speed, server load, etc.

Also, although most of the times, in theory, you won't be charged since you are going to use Cloud 9 as a free-tier user, don't feel surprised if you receive a bill from AWS.

Around a couple of years ago, Cloud9 (which used to be owned by an independent vendor) was acquired by Amazon Web Services (AWS). Therefore now, in order to use Cloud9, you need to go through three steps as summarized below:

  • Create an AWS account (A credit card is needed)
  • Create an AWS Identity and Access Management (IAM) group and user
  • Create an AWS Cloud9 environment

Then move on to Part 4 below.

3. Get Ready for Github and Heroku on your own Linux/Mac computer

3.1. Github

  • Go to https://help.github.com/articles/set-up-git/. Follow the three steps to download, install, and set up Git on your computer.

  • Go to https://help.github.com/articles/generating-ssh-keys. Follow the steps there to create your HTTPS/SSH key and add it to GitHub. This is to authenticate the connection between the Git repositories on your local computer and GitHub as your remote host.

    If the -K (in upper case) option for the ssh-add command is not recognized, change it to -k (in lower case).

    If the pbcopy < ~/.ssh/id_rsa.pub command for copying the public SSH key is not working, just use cat ~/.ssh/id_rsa.pub and copy the output by yourself.

3.2. Heroku

  • Go to https://devcenter.heroku.com/articles/heroku-cli to download and install the Heroku CLI (Command Line Interface).

  • In a terminal window, use the following commands to log in and add your SSH key to Heroku:

    heroku login
heroku keys:add

    This is to authenticate the connection between your local Git repositories and Heroku as the cloud application platform where you build and deploy your Full-Stack Web apps.

3.3. Code editor

There are a number of awesome code editors for you to choose from. To list just a few: Visual Studio Code, Sublime Text, Atom, Textmate, Brackets, and of course Vim.

Although it is optional, but I HIGHLY RECOMMEND the following add-ons for a more efficient coding experience. The steps to install them vary depending on the code editor that you use. Below, I am using sublime as an example.

  • "babel-sublime" for ES6+ JavaScript with React JSX syntax extensions. Just go to https://github.com/babel/babel-sublime and follow the instruction to download the package and set it up.

  • "Prettier" -- a fantastic editor add-on to auto-format your JS code. Just go to https://github.com/prettier/prettier, scroll down to "Documentation", "Install", then "Editor Integration", and follow the instructions to install it. Then, auto-format is just one key away (e.g. on Mac, Option+Command+f).

    Make sure you go the Preferences > Package Settings > JsPrettier menu and turn "auto_format_on_save" on. That way, your code will match mine, which will make our lives much easier when doing code debugging together.

4. Get Ready for Github and Heroku on AWS Cloud9

4.1. Git

  • Git comes with every Cloud9 environment so you don't need to install it. However, you still need to authenticate the connection between your local Git repositories on Cloud9 and GitHub.

    Setup the following two Git globals:

    git config --global user.name "your name"
git config --global user.email "your email"

    Then, follow the instructions at https://help.github.com/articles/generating-ssh-keys.

    If the -K (in upper case) option for the ssh-add command is not recognized, change it to -k (in lower case).

    Also, if the pbcopy < ~/.ssh/id_rsa.pub command for copying the public SSH key is not working, just use cat ~/.ssh/id_rsa.pub and copy the output by yourself.

4.2. Get ready for Heroku

  • Install the Heroku CLI (Command Line Interface) first.

    cd
wget https://cli-assets.heroku.com/heroku-cli/channels/stable/heroku-cli-linux-x64.tar.gz -O heroku.tar.gz
tar -xvzf heroku.tar.gz
mkdir -p /usr/local/lib /usr/local/bin
sudo mv  heroku-cli-v6.99.0-ec9edad-linux-x64 /usr/local/lib/heroku
sudo ln -s /usr/local/lib/heroku/bin/heroku /usr/local/bin/heroku
rm heroku.tar.gz

    Note: for the sudo mv command above, replace "heroku-cli-..." with the actual name of your Heroku CLI folder.

  • Then use the following commands to log in Heroku and add your SSH public key to Heroku:

    heroku login
heroku keys:add

    This is to authenticate the connection between your local Git repositories on AWS Cloud9 and Heroku as the cloud application platform where you build and deploy your Full-Stack Web apps.

4.3. Editor

A full-fledged code editor comes with every Cloud9 environment so you don't need to install a separate one.
