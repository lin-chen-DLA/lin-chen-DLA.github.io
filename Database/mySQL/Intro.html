Introduction
Data Types
  • Numeric
    • INT
    • FLOAT(M, D)
      • M, display length
      • D, number of decimals
      • a floating-point data type that stores approximate values
      • don't require precise decimal values
      • FLOAT(10, 2) # 10, lengh, 2 is the number of decimals
    • DOUBLE(M, D)
      • a floating-point data type that stores larger and more precise values than FLOAT
    • DECIMAL(M, D)
      • A fixed-precision data type
      • Store values that require absolute precision
  • Date and Time Types
    • DATETIME
    • TIMESTAMP
  • String
    • CHAR(M), A fixed-length string
    • VARCHAR(M), A variable-length string
    • BLOB or TEXT, a field with a maximum length of 65535 characters
    • Transaction
  • A sequential group of database manipulation operations
  • Perform as if it were one single work unit
  • ACID
    • Atomicity, ensures that all operations within the work unit are completed successfully
    • Consistency, ensures that the database properly changes states upon a successfully committed transaction
    • Isolation, enables transactions to operate independently on and transparent to each other
    • Durability, ensures that the result or effect of a committed transaction persists in case of a system failure
  • COMMIT and ROLLBACK
    • When a successful transaction is completed, the COMMIT command should be issued so that the changes to all involved tables will take effect
    • If a failure occurs, a ROLLBACK command should be issued to return every table referenced in the transaction to its previous state
