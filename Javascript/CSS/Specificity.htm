Specificity
Specificity Hierarchy
  • If there are two or more conflicting CSS rules that point to the same element, the browser follows some rules to determine which one is most specific and therefore wins out
  • (+1000) Inline styles, an inline style is attached directly to the element to be styled
  • (+100) IDs, an ID is a unique identifier for the page elements
  • (+10) Classes, attributes and pseudo-classes, includes .classes, [attributes] and pseudo-classes
  • (+1) Elements and pseudo-elements, includes element names and pseudo-elements
    • <!DOCTYPE html>
<html lang="en">
<head>
	<meta charset="UTF-8">
	<title>Document</title>
	<style>
		div {color: red;} //1
		.section {color: blue;} //10
	</style>
</head>
<body>
		<div class = "section">
			Hello World! <!-- blue color -->
		</div>
	
</body>
</html>
    Specificity Rules
  • Equal specificity: the latest rule counts, if the same rule is written twice into the external style sheet, then the lower rule in the style sheet is closer to the element to be styled
  • Contextual selectors are more specific than a single element selector, the embedded style sheet is closer to the element to be styled
    • <!DOCTYPE html>
<html lang="en">
<head>
	<meta charset="UTF-8">
	<title>Document</title>
	<style>
		.section {color: blue;}
		.section {color: red;}
	</style>
</head>
<body>
		<div class = "section">
			Hello World!
		</div>
	
</body>
</html>
  • The universal selector and inherited values have a specificity of 0 - *, body * and similar have a zero specificity. Inherited values also have a specificity of 0
    • <!DOCTYPE html>
<html lang="en">
<head>
	<meta charset="UTF-8">
	<title>Document</title>
	<style>
		#content {color: red;}
		.section {color: blue;}
	</style>
</head>
<body>
	<div id = "content">
		<div class = "section">
			Hello World! <-- blue color -->
		</div>
	</div>
	
</body>
</html>
