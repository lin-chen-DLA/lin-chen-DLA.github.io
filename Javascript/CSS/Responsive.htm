Always Design for Mobile First
  • Mobile First means designing for mobile before designing for desktop or any other device
  • Instead of changing styles when the width gets smaller than a threshold, we should change the design when the width gets larger than the threshold
    • One Breakpoint

    <!DOCTYPE html>
<html lang="en">
<head>
	<meta charset="UTF-8">
	<title>Document</title>
	<style>
		* {box-sizing: border-box;}
		div {border: 1px solid red;}
		[class*="col-"] {width: 100%; float:left; padding: 10px;}

		@media only screen and (min-width: 600px)
		/* Tablet Devices */
		{
			.col-s-1 {width: 20%;}
			.col-s-2 {width: 40%;}
			.col-s-3 {width: 60%;}
			.col-s-4 {width: 100%;}
		}

		@media only screen and (min-width: 768px)
		/* Desktop Devices */
		{
			.col-1 {width: 20%;}
			.col-2 {width: 40%;}
			.col-3 {width: 60%;}
			.col-4 {width: 100%;}
		}
	</style>
</head>
<body>
	<div class = "col-s-2 col-1">Colume 1</div>
	<div class = "col-s-3 col-2">Colume 2</div>
	<div class = "col-s-4 col-2">Colume 3</div>
</body>
</html>
    Typical Device Breakpoints
  • Phones, portrait tablets and large phones, 600px
  • landscape tablets, 768px
  • laptops/desktops, 992px
  • large laptops and desktops, 1200px
