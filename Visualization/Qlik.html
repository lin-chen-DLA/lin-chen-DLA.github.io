    Data Model
  • Refers to the structure and organization of data within the Qlik platform
  • Data is loaded into an in-memory data model which is similar to a relational database
  • Identifies common fields between tables and establishes associations between them
  • Associations are based on shared values within the fields. Bi-directional, selections made in one table can affect data in related tables and vice versa. Likes full outer joined tables
  • Option 1. Data Manager
    • 1. Load data tables from local
    • 2. Build associations
  • Option 2. Data Load Editor
    • 1. Create database connection or local file connection
    • 2. Load tables with scripts
      • LIB CONNECT TO 'connection_name'

    • Search
  • Text search
    • Normal search, strings that match the search string are displayed
      • NEAR # NEAR BAY, NEAR OCEAN
"NEAR BAY" # NEAR BAY
NEAR Bay # NEAR BAY, NEAR OCEAN, values that contain any of the search strings
+OCEAN NEAR # NEAR OCEAN, <1H OCEAN, strings with a plus sign must be included
NEAR -BAY # NEAR OCEAN, removes all values containing this string with a minus sign
    • Wildcard search
      • # *, in the place of zero or more characters
N* # NEAR BAY, NEAR OCEAN

# ?, in the place of a single character
N?ar* # NEAR BAY, NEAR OCEAN

# ^, beginning of a work
*^Ba* # NEAR BAY
  • Numeric search 
      > 100
> 100 < 200 # and
  • Fuzzy search 
      # returns a list of values which may not be identical to the search input
~neay ocean # NEAR OCEAN, <H OCEAN, NEAR BAY
  • Expression search 
      # Search for values across all fields associated with the search field
# An expression search always begins with an equals sign =
# within the filter panel of ocean_proximity
=avg(total_rooms) > 2500 # <1H OCEAN, INLAND, NEAR OCEAN
  • Compound search 
      # use multiple search operators in a single compound search
# enclose the search within a set of parentheses
(N*|*land) # INLAND, ISLAND, NEAR BAY, NEAR OCEAN
