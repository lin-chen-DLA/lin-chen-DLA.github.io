Amazon IAM
What is Identity and Access Management (IAM)
  • Safeguards accesses to AWS services and resources and to create and manage users and groups and user permissions to grant or deny access to AWS services
    • Working of IAM
  • Principal
    • An action on an AWS resource can be performed by a principal
    • A user or a role can be a principal
  • Authentication
    • A process of confirming identity of the principal trying to access an AWS product
    • To authenticate from console, need credentials or required keys
  • Request
    • When a principal attemps to access the AWS console, API or CLI, send a request to AWS
  • Authorizations
    • Here IAM uses information from the request context to check for matching policies and determine whether to allow or deny the request
  • Actions
    • After authentication and authorization of the request, AWS approves the action
    • Using this, either edit, delete, or create a resource
  • Resources
    • A set of actions can be performed in a related resource of the AWS account
    • Components of IAM
  • Users
    • IAM users are AWS entities that represent the person or application that interacts with AWS
    • AWS users have a name and credentials
    • The root user of an AWS account is not the same as an IAM user with administrator access
  • Groups
    • A set of IAM users is referred to as an IAM user group
    • Allow specify permissions for many users
  • Roles
    • Collection of policies
    • There are no long-term credentials connected with a role. When given a role, the role has a temporary security credentials for the duration of your role session
  • Policies
    • Created and linked to IAM identities or AWS resources
    • Permission to resources
    • The policy permissions decide whether the request is approved or rejected
    • Types
      • Customer managed
      • AWS managed
      • AWS managed - job function
