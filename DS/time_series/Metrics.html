Evaluation¶

$R^{2}$¶

$$ R^{2} = \frac{\sum_{i = 1}^{n}|y_{i} - f_{i}|}{\sum_{i = 1}^{n}|y_{i} - \bar{y}|}$$

  • $y_{i}$, true value
  • $f_{i}$, predicted value
  • $\bar{y} = \sum_{i = 1}^{n} y_{i}$
  • Expresses the proportion of the variance in the dependent variable that is predictable from the independent variables, 1 means perfect relationship, 0 means no relationship
  • $R^{2}$ is not an error measure metric
In [ ]:
from sklearn.metrics import r2_score
r2_score(y_test, y_pred) # 0.92

Mean Absolute Error (MAE)¶

$$ MAE = \frac{1}{n}\sum_{i = 1}^{n}|y_{i} - f_{i}|$$

  • $y_{i}$, true value
  • $f_{i}$, predicted value
  • Range
    • $0 - \infty$, the lower the value the better and 0 means the model is perfect
  • Pros
    • Easy to interpret, suitable for continuous variables, and less sensitive to outliers
  • Cons
    • Need to compare with other MAE to check if this MAE is good/bad
In [ ]:
# univariate
from sktime.performance_metrics.forecasting import MeanAbsoluteError

mae(y_test, y_pred)

from sktime.performance_metrics.forecasting import mean_absolute_error
mean_absolute_error(y_test, y_pred)

# multivariate

y_true_m = np.array([[0.5, 1], [-1, 1], [7, -6]]) # 3*2
y_pred_m = np.array([[0, 2], [-1, 2], [8, -5]]) # 3*2

# ‘raw_values’, returns an error for each output
mae = MeanAbsoluteError(multioutput='raw_values')
mae(y_true_m, y_pred_m) # 0.5, 1

# ‘uniform_average’, return an error for all outputs with uniform weight
mae = MeanAbsoluteError(multioutput='uniform_average')
mae(y_true_m, y_pred_m) # 0.75

# # ‘uniform_average’, return an error, each output has different weight
mae = MeanAbsoluteError(multioutput=[0.2, 0.8])
mae(y_true_m, y_pred_m) # 0.9

Mean Squared Error (MSE)¶

$$ MSE = \frac{1}{n}\sum_{i = 1}^{n}(y_{i} - f_{i})^2$$

  • $y_{i}$, true value
  • $f_{i}$, predicted value
  • Range
      • $0 - \infty$, the lower the value the better and 0 means the model is perfect
  • Pros
    • gives more weight to larger errors ensuring no outlier predictions with large errors
  • Cons
    • sensitivity to outliers, magnify the error from a single bad prediction
In [ ]:
from sktime.performance_metrics.forecasting import MeanSquaredError
mse = MeanSquaredError()
mse(y_test, y_pred)

from sktime.performance_metrics.forecasting import mean_squared_error
mean_squared_error(y_test, y_pred)

Root Mean Squared Error (RMSE)¶

$$ RMSE = \sqrt{MSE}$$

  • Range
      • $0 - \infty$, the lower the value the better and 0 means the model is perfect
  • Pros
    • penalizes larger errors more severely
    • same units as original data
  • Cons
    • The magnitude of RMSE is directly related to the scale of the dependent variable making it difficult to compare RMSE values across different datasets or models with different target scales
In [ ]:
from sktime.performance_metrics.forecasting import MeanSquaredError
rmse = MeanSquaredError(square_root=True)
rmse(y_test, y_pred)

from sktime.performance_metrics.forecasting import mean_squared_error
mean_squared_error(y_test, y_pred, square_root=True)

Mean Absolute Scaled Error (MASE)¶

$$MASE = \frac{\frac{1}{n}\sum_{i = 1}^{n}|y_{i} - f_{i}|}{\frac{1}{n-1}\sum_{i=2}^{n}|y_{i} - y_{i-1}|}$$

  • $y_{i}$, true value
  • $f_{i}$, predicted value
  • Compare the model’s error to a naive forecast, numerator is the MAE, denominator is the MAE of the naive forecast. The naive forecast is generated by equating the current forecast to the output from the last time step
  • Range
    • $0 - \infty$, the lower the value the better and 0 means the model is perfect
    • if MASE < 1 indicates that the forecasting model performs better than the naive forecast; if MASE > 1, the forecast is worse than the naive forecast
  • Pros
    • scale-independent
    • never gives undefined or infinite values
    • robust to Outliers
  • Cons
    • the MASE values may not be meaningful if the naive forecast is not appropriate for the data
In [ ]:
from sktime.performance_metrics.forecasting import MeanAbsoluteScaledError
mase = MeanAbsoluteScaledError()
mase(y_test, y_pred, y_train = y_train)

from sktime.performance_metrics.forecasting import mean_absolute_scaled_error
mean_absolute_scaled_error(y_test, y_pred, y_train = y_train)

Mean Squared Scaled Error (MSSE)¶

$$MSSE = \frac{\frac{1}{n}\sum_{i = 1}^{n}(y_{i} - f_{i})^2}{\frac{1}{n-1}\sum_{i=2}^{n}(y_{i} - y_{i-1})^2}$$

  • $y_{i}$, true value
  • $f_{i}$, predicted value
  • Range
    • $0 - \infty$, the lower the value the better and 0 means the model is perfect
    • if MSSE < 1 indicates that the forecasting model performs better than the naive forecast; if MSSE > 1, the forecast is worse than the naive forecast
  • Pros
    • scale independence
  • Cons
    • sensitive to the choice of forecasting model and naive benchmark

Geometric Mean Absolute Error (GMAE)¶

$$GMAE = (\prod_{i=1}^{n}|y_{i}-f_{i}|)^{1/n}$$

  • $y_{i}$, true value
  • $f_{i}$, predicted value
  • Range
    • $0 - \infty$, the lower the value the better
  • Pros
    • places greater emphasis on large errors compared to traditional error metrics like Mean Absolute Error (MAE) or Root Mean Squared Error (RMSE)
    • robust to outliers
  • Cons
    • when any prediction is accurate, meaning that the absolute error between the predicted values and the actual values is zero for all observationsis zero

Geometric Mean Squared Error (GMSE)¶

$$GMSE = (\prod_{i=1}^{n}(y_{i}-f_{i})^2)^{1/n}$$

  • $y_{i}$, true value
  • $f_{i}$, predicted value
  • Range
    • $0 - \infty$, the lower the value the better
  • Pros
    • Suitability for skewed data
    • robust to outliers
  • Cons
    • assumption of squared error

Mean Absolute Percentage Error (MAPE)¶

  • Asymmetric $$ MAPE = \sum_{i = 1}^{n} |\frac{y_{i} - f_{i}}{y_{i}}|$$
  • Symmetric $$ sMAPE = \frac{2}{n} \sum_{i = 1}^{n} \frac{|y_{i} - f_{i}|}{|y_{i}|+|f_{i}|}$$
  • $y_{i}$, true value
  • $f_{i}$, predicted value
  • output non-negative floating point which is in fractional units rather than percentage
  • Range
    • MAPE = 0%, perfect forecast accuracy
    • 0% < MAPE < 100%, indicates forecast accuracy between perfect and no accuracy
    • MAPE = 100%, indicates no forecast accuracy
    • MAPE > 100%: indicates poor forecast accuracy
  • Pros
    • provides a percentage-based measure of forecast accuracy
    • scale independence
  • Cons
    • MAPE can encounter division by zero issues
In [ ]:
from sktime.performance_metrics.forecasting import MeanAbsolutePercentageError

smape(y_test, y_pred)

Mean Squared Percentage Error (MSPE)¶

  • Asymmetric $$ MSPE = \sum_{i = 1}^{n} (\frac{y_{i} - f_{i}}{y_{i}})^2$$
  • Symmetric $$ sMSPE = \frac{1}{n} \sum_{i = 1}^{n} (\frac{y_{i} - f_{i}}{(y_{i}+f_{i})/2})^2$$
  • $y_{i}$, true value
  • $f_{i}$, predicted value
  • output non-negative floating point which is in fractional units rather than percentage
  • Range
    • MSPE = 0, indicates perfect forecast accuracy
    • MSPE > 0, indicates the presence of forecast errors
  • Pros
    • Sensitive to large errors
    • scale independence
  • Cons
    • can be excessively influenced by small actual value
    • can give excessive weight to outliers
    • can encounter division by zero issues

Median Absolute Error (MedAE)¶

$$ MedAE = median(|y_{1}-\hat{y}_1|, ..., |y_{n}-\hat{y}_n|)$$

  • $y_{i}$, true value
  • $\hat{y}_{i}$, predicted value
  • A non-negative floating point; the best value is 0.0
  • Pros
    • robust to outliers

Median Absolute Scaled Error (MedASE)¶

$$ MedASE = \frac{median(|y_{1}-\hat{y}_1|, ..., |y_{n}-\hat{y}_n|)}{{\frac{1}{n-1}\sum_{i=2}^{n}|y_{i} - y_{i-1}|}}$$

  • $y_{i}$, true value
  • $\hat{y}_{i}$, predicted value
  • Pros
    • more robust to error outliers than MASE
    • can be used to compare forecast methods on a single series or between series
    • will not give infinite or undefined values unless the training data is a flat timeseries
