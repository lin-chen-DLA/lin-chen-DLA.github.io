Jira Core
  • Administrators
    • create and configure a project
    • are responsible for the configuration of Jira Core
    • control the initial set-up, the look and feel, and the configuration of project workflows, issues, and generally what the end user will see and be able to interact with
    • 1. set up an instance of Jira Core
      • Sign Up Jira Cloud
      • lin.chen@ieee.org
      • lin-chen.atlassian.net
    • 2. add new users
      • Administration -> User Management
    • 3. create a project and customize it
      • project is a container that holds issues
      • Issues are the packets of work required within a project
      • Administration -> Projects
    • 4. manage permissions
      • Global permissions, affect all projects in Jira Core, can only be assigned to groups
      • Project permissions, affect a single project in Jira Core, can be assigned to groups, users and project-specific roles
      • Create a project role
        • Adminstration -> System
      • Configure a new permission scheme
        • Administration -> Issues -> Permission Schemes
      • Associate the scheme with a new project
        • Project settings -> Permissions
  • Project Administrators
    • customize your project
    • add users to your project
  • Users
    • access projects
    • work with issues
    • search for issues
